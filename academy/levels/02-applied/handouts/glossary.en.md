# Glossary — Level 2 Terms in Plain Language

The short, no-jargon dictionary for Level 2. It carries forward the Level 1 basics and adds the new
terms you need to build a template kit. Definitions are written for everyday business teams — if you
only read one column, read "Plain meaning."

---

| Term | Plain meaning | Quick example |
|---|---|---|
| **Prompt** | The instruction you give the AI — your request, in your own words. | "Write a 4-sentence reply to this complaint." |
| **Prompt formula** | The Role · Task · Context · Format recipe from Level 1 for writing clear prompts. | The base every template is built on. |
| **Template** | A reusable prompt with the changing parts marked as `{variables}` and the rest locked, so anyone reuses it the same way. | A saved "carrier delay reply" prompt with `{order number}`. |
| **Variable (placeholder)** | The blank you fill in each time, written in `{curly braces}`. It marks what changes. | `{customer name}`, `{date}`, `{tone}`. |
| **Locked frame** | The parts of a template that stay the same every run: role, tone, language, format. | "Always a warm, under-90-word email in Spanish." |
| **Template card** | The standard way to save a template: name, task, the template, when-to-use, check-before-sending, language, owner. | One entry in your team kit. |
| **Template kit** | A shared, organized library of your team's templates — the Level 2 deliverable. | A doc or wiki page everyone reuses. |
| **Tone** | The feel of the writing — warm, formal, concise, upbeat. You set it in the template so results stay consistent. | "Warm and professional" vs. "short and formal." |
| **Iteration (refine)** | Improving a result by replying in the same chat instead of starting over. | "Make it shorter," "add an example." |
| **Refine, don't redo** | The habit of steering an almost-right answer with one instruction rather than rewriting from scratch. | Fix the one thing that's off. |
| **Model choice** | Deciding which assistant or mode to use: a fast one for everyday text, a "most capable" one for long or high-stakes work. | Quick email → fast; big careful summary → most capable. |
| **AI model** (or "model") | The trained "engine" behind the tool. Different models trade speed for capability. | Like different brands of engine. |
| **Summary** | A shorter version of a long text that keeps the key points and next steps. Always verify it kept the facts. | Email thread → 5 bullets + next steps. |
| **Spreadsheet / table helper** | Using AI to turn messy text into a clean table or to explain a formula — which you then rebuild and check in the real sheet. | "Make a table with these columns." |
| **Image prompt** | A clear description that tells the AI what picture to create: subject, style, colors, orientation, any text. | "Horizontal banner, flat style, brand red accents." |
| **Hallucination** | When AI states something false but sounds confident. Always verify facts and figures. | It invents a statistic that isn't real. |
| **Human-in-the-loop** | A person reviews, corrects, and owns the AI's output before it's used — the check-before-sending step. | You edit the draft and send it under your name. |
| **PII** | Personally Identifiable Information — data that identifies a person (name, address, ID, card number). Don't paste into public tools without policy. | A customer's full name + account number. |
| **PHI** | Protected Health Information — health data tied to a person. Extra-sensitive; handle under approved policy only. | A patient's name + diagnosis. |
| **Confidential data** | Private business info: contracts, pricing, trade secrets, internal plans. Treat like PII. | A draft contract or salary list. |
| **Responsible AI** | Using AI safely and ethically: protect data, verify facts, review output, disclose when required. | The 4 checks on your cheat-sheet. |
| **Owner** | The person who keeps a template card (or the whole kit) current and correct. | "Owner: Dispatch lead." |
