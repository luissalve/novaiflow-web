# Level 1 Project Brief — 3 Prompts for Your Role

This is your Level 1 deliverable: **three useful prompts, written for your real job, ready to reuse.**
It's small on purpose. By the end of the 6‑minute project block you'll have your **first AI asset** —
and it becomes the seed of your team's template kit in Level 2.

You already wrote prompt #1 in the workshop. Here you add two more, polish all three, and save them in
one simple format anyone on your team can pick up and use.

---

## Objective

Capture **three prompts you'll actually use** at work, each built with the **Role · Task · Context ·
Format** formula, and each written so a teammate could reuse it next week.

You'll know you're done when:

- You have **3 prompts**, each for a different real task you do often.
- Each prompt is **specific** (clear role, task, context, format).
- Each prompt is **reusable** (uses `{placeholders}` for the parts that change).
- Each prompt is **safe** (no sensitive data baked in; a person reviews the output).

---

## What to submit — the exact format

Submit one short document (or filled worksheet) with this table. Keep prompt #1 from your workshop and
add #2 and #3.

| # | Task | Prompt | When I use it |
|---|---|---|---|
| 1 | *(short name of the task)* | *(the full prompt, with `{placeholders}`)* | *(the situation that triggers it)* |
| 2 | | | |
| 3 | | | |

**Field notes:**

- **Task** — a short label so you can find it later (e.g., "Carrier delay reply").
- **Prompt** — the complete, ready‑to‑paste prompt. Use `{curly braces}` for anything you fill in each time.
- **When I use it** — the trigger, so a teammate knows *when* to reach for it.

### Prompt template (use this skeleton for each row)

```text
Act as {role}.
{Task — one clear verb} this for me.
Context: {relevant, non-sensitive details — use {placeholders} for what changes}.
Format: {length}, {tone}, in {language}, as {bullets / short paragraph / table}.
```

## Two filled examples (different roles)

### Example A — Logistics / Customer service

| # | Task | Prompt | When I use it |
|---|---|---|---|
| 1 | Carrier delay reply | *(see prompt below)* | A customer asks why their shipment is late |

```text
Act as a friendly bilingual customer-service rep for a freight company.
Draft a reply to a customer whose shipment is delayed.
Context: order {order number}, was due {original date}, new ETA {new date}, reason {reason}.
Format: under 90 words, warm and professional, in {English/Spanish}, end with one clear next step.
```

### Example B — Clinic / Front desk

| # | Task | Prompt | When I use it |
|---|---|---|---|
| 1 | No‑show follow‑up | *(see prompt below)* | A patient misses a routine appointment |

```text
Act as a warm bilingual front-desk coordinator for a dental clinic.
Draft a no-show follow-up message to {patient first name}.
Context: missed a {appointment type}; offer two reschedule options: {option 1}, {option 2}.
Format: under 60 words, friendly and respectful, in {language}, ready to send by text. No medical advice.
```

> **Note on both examples:** the prompts use `{placeholders}` instead of real order numbers or patient
> names. That's what makes them safe to reuse and safe to share. Never paste real patient or client
> data into a public tool without an approved policy.

## Acceptance criteria

Your three prompts pass if every box is ticked:

### Specific

- [ ] Each prompt names a **Role** ("Act as…").
- [ ] Each prompt has a **Task** with one clear verb (draft, summarize, translate, reply…).
- [ ] Each prompt gives **Context** the assistant needs.
- [ ] Each prompt states a **Format** (length, tone, language, structure).

### Reusable

- [ ] The parts that change each time are written as `{placeholders}`, not hard‑coded.
- [ ] A teammate could read the prompt and know exactly when and how to use it.
- [ ] Each of the three covers a **different** real task you do often.

### Safe (Responsible AI)

- [ ] **No sensitive data** (PHI, PII, confidential) is baked into the prompt — placeholders only.
- [ ] The "When I use it" note assumes **a person reviews** the output before it's used.
- [ ] Any facts or figures the AI produces are flagged in your mind as **"verify before sending."**
- [ ] If your role requires disclosure or confidentiality, the prompt respects it.

> If any box is unchecked, refine that prompt before submitting. Use the workshop coaching tips — most
> fixes are "add the missing part of the formula" or "replace a real detail with a `{placeholder}`."
