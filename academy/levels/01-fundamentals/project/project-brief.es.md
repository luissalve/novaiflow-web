# Brief del Proyecto · Nivel 1 — 3 prompts para su rol

Este es su entregable del Nivel 1: **tres prompts útiles, escritos para su trabajo real, listos para
reutilizar.** Es pequeño a propósito. Al terminar el bloque de proyecto de 6 minutos tendrá su
**primer activo de IA** — y se convierte en la semilla del kit de plantillas de su equipo en el Nivel 2.

Usted ya escribió el prompt #1 en el workshop. Aquí agrega dos más, pule los tres, y los guarda en un
formato simple que cualquiera de su equipo puede tomar y usar.

---

## Objetivo

Capturar **tres prompts que de verdad usará** en el trabajo, cada uno construido con la fórmula
**Rol · Tarea · Contexto · Formato**, y cada uno escrito para que un colega pueda reutilizarlo la
próxima semana.

Sabrá que terminó cuando:

- Tenga **3 prompts**, cada uno para una tarea real distinta que hace seguido.
- Cada prompt sea **específico** (rol, tarea, contexto y formato claros).
- Cada prompt sea **reutilizable** (use `{variables}` para las partes que cambian).
- Cada prompt sea **seguro** (sin datos sensibles incrustados; una persona revisa el resultado).

---

## Qué entregar — el formato exacto

Entregue un documento corto (o la hoja de trabajo llena) con esta tabla. Conserve el prompt #1 de su
workshop y agregue el #2 y el #3.

| # | Tarea | Prompt | Cuándo lo uso |
|---|---|---|---|
| 1 | *(nombre corto de la tarea)* | *(el prompt completo, con `{variables}`)* | *(la situación que lo dispara)* |
| 2 | | | |
| 3 | | | |

**Notas de cada campo:**

- **Tarea** — una etiqueta corta para encontrarlo después (p. ej., "Respuesta por retraso de transportista").
- **Prompt** — el prompt completo, listo para pegar. Use `{llaves}` para todo lo que llena cada vez.
- **Cuándo lo uso** — el disparador, para que un colega sepa *cuándo* recurrir a él.

### Plantilla del prompt (use este esqueleto para cada fila)

```text
Actúa como {rol}.
{Tarea — un verbo claro} esto para mí.
Contexto: {detalles relevantes y no sensibles — use {variables} para lo que cambia}.
Formato: {largo}, {tono}, en {idioma}, como {viñetas / párrafo corto / tabla}.
```

## Dos ejemplos llenos (roles distintos)

### Ejemplo A — Logística / Atención al cliente

| # | Tarea | Prompt | Cuándo lo uso |
|---|---|---|---|
| 1 | Respuesta por retraso de transportista | *(ver prompt abajo)* | Un cliente pregunta por qué su envío está demorado |

```text
Actúa como un representante bilingüe y amable de atención al cliente de una empresa de freight.
Redacta una respuesta para un cliente cuyo envío está demorado.
Contexto: pedido {número de pedido}, vencía {fecha original}, nuevo ETA {nueva fecha}, motivo {motivo}.
Formato: menos de 90 palabras, cálido y profesional, en {inglés/español}, termina con un siguiente paso claro.
```

### Ejemplo B — Clínica / Recepción

| # | Tarea | Prompt | Cuándo lo uso |
|---|---|---|---|
| 1 | Seguimiento por inasistencia | *(ver prompt abajo)* | Un paciente falta a una cita de rutina |

```text
Actúa como coordinador(a) bilingüe y cálido(a) de recepción de una clínica dental.
Redacta un mensaje de seguimiento por inasistencia para {nombre del paciente}.
Contexto: faltó a una {tipo de cita}; ofrece dos opciones para reagendar: {opción 1}, {opción 2}.
Formato: menos de 60 palabras, amable y respetuoso, en {idioma}, listo para enviar por mensaje. Sin consejo médico.
```

> **Nota sobre ambos ejemplos:** los prompts usan `{variables}` en lugar de números de pedido o
> nombres de pacientes reales. Eso es lo que los hace seguros para reutilizar y seguros para
> compartir. Nunca pegue datos reales de pacientes o clientes en una herramienta pública sin una
> política aprobada.

## Criterios de aceptación

Sus tres prompts aprueban si cada casilla está marcada:

### Específico

- [ ] Cada prompt nombra un **Rol** ("Actúa como…").
- [ ] Cada prompt tiene una **Tarea** con un verbo claro (redacta, resume, traduce, responde…).
- [ ] Cada prompt da el **Contexto** que el asistente necesita.
- [ ] Cada prompt indica un **Formato** (largo, tono, idioma, estructura).

### Reutilizable

- [ ] Las partes que cambian cada vez están escritas como `{variables}`, no fijas.
- [ ] Un colega podría leer el prompt y saber exactamente cuándo y cómo usarlo.
- [ ] Cada uno de los tres cubre una tarea real **distinta** que hace seguido.

### Seguro (IA responsable)

- [ ] **Sin datos sensibles** (PHI, PII, confidenciales) incrustados en el prompt — solo variables.
- [ ] La nota "Cuándo lo uso" asume que **una persona revisa** el resultado antes de usarlo.
- [ ] Cualquier dato o cifra que produzca la IA queda marcado en su mente como **"verificar antes de enviar."**
- [ ] Si su rol exige divulgación o confidencialidad, el prompt la respeta.

> Si alguna casilla queda sin marcar, refine ese prompt antes de entregar. Use los tips de
> acompañamiento del workshop — la mayoría de los arreglos son "agregue la parte que falta de la
> fórmula" o "reemplace un dato real por una `{variable}`."

más rápido se arma el kit de su equipo después.
